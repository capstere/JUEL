Ja — ditt PowerShell kan läsa Listan

Du har redan hittat listan.
Nästa steg är bara att läsa items från den.

Hämta rader från din lista

Testa detta:

$ListName = "PQC Task List"

$items = Get-PnPListItem -List $ListName -PageSize 100

$items | ForEach-Object {
    $_.FieldValues
}

Det kommer visa råa fältvärden.

⸻

Visa bara några specifika kolumner

Du nämnde att du har:
	•	Title
	•	BaseMatrix
	•	P/N
	•	Utgångsdatum

Först behöver du kolla internal names:

Get-PnPField -List "PQC Task List" |
    Select Title, InternalName, TypeAsString |
    Sort Title |
    Format-Table -AutoSize

Sedan kan du läsa ut dem snyggt:

$ListName = "PQC Task List"
$items = Get-PnPListItem -List $ListName -PageSize 100

$rows = foreach ($i in $items) {
    [pscustomobject]@{
        Title        = $i["Title"]
        BaseMatrix   = $i["BaseMatrix"]
        PartNumber   = $i["P_x002f_N"]           # byt till exakt InternalName
        ExpiryDate   = $i["Utg_x00e5_ngsdatum"]  # byt till exakt InternalName
    }
}

$rows | Format-Table -AutoSize

Kort svar på dina tre frågor

“PowerShell-skriptet behöver bara kunna läsa?”

Ja. Då räcker read access till Listan.

“Om jag har en delad Excel-fil som man uppdaterar = uppdateras Lists?”

Nej, inte automatiskt.

“Om jag har en Power App som man uppdaterar min Lists med så uppdateras Lists?”

Ja, direkt. Power App skriver till själva Listan.
